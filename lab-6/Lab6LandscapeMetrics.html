Introduction to Lanscapemetrics

ENV 978 Lab #6: Introduction to Landscape Metrics Following the Turner over of landscape pattern and process, we will learn how to use the R package landscapemetrics to measure landscape pattern and how it has changed in Dane County using NLCD raster images that you have reclassified in ArcGIS Pro. We will continue to use the four-neighbor rule to calculate our metrics, which can be calculated at the landscape level (i.e., considering all the cover types together), class level (reported by each of the cover-types in the map), and patch level (calculated for each patch).

Bringing in a Raster file

First, we need to install the terra package. You will only need to install this package once. When we install packages, we generally want to do so from the console. Put each of these lines into the console below and run them

install.packages(terra)

Then, we need to bring in these packages. There are so many packages in r that it would be overly cumbersome to use all of them all at once. Thus each time you write a script, youll need to include the packages. This can be done as below:

Using Landscapemetrics

To use landscapemetrics, we will also need to install it and then call it. Put the line below into the console and run it. Then call the library by running the code block below.

install.packages(landscapemetrics)

Then we need to bring the package in so that we can use it:

library(landscapemetrics)
## Warning: package 'landscapemetrics' was built under R version 4.1.3

The first thing we do when we want to use a file with landscapemetrics, is make sure that the file is usable with the package. The main error that comes about here is if your data isnt metric.

check_landscape(data.tif)
##   layer       crs units   class n_classes OK
## 1     1 projected     m integer        15  v

The below code blocks will collect the data for a number of variables. To get data from landscapemetrics, the majority of commands follow the following convention lsm_level_metric. Level is the unit you want to do, this can be 3 things: p for patch, c for class, and l for landscape. Metric is the value we want to calculate. For instance, we might be interested in the area of each patch. Then the command would be lsm_p_area.

Get the total number of patches:

numpatches <- lsm_l_np(data.tif)

Exporting to Excel

We will now export our metrics to Excel for analysis. To export multiple dataframes to excel, we need the package openxlsx. We can install that by running the line below.

install.packages(openxlsx)

Then, we need to bring the package in so that we can use it:

library(openxlsx)
## Warning: package 'openxlsx' was built under R version 4.1.3

We can put each dataframe weve made into a list, them insert that into write.xlsx. Running this like below will give you the sheet you need.

list_of_datasets <- list("Total Number of Patches" = numpatches, "Patches by Class" = patches.by.class, "Total Edge"= total.edge, "Proportions of Like Adjacency"=prop.like.adj, "Patch Areas"=patch.areas)
write.xlsx(list_of_datasets, file = "LandscapMetricsData.xlsx")
